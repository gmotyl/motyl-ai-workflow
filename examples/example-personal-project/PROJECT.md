## Overview

This is an example personal project showing the folder structure and patterns used in motyl-ai-workflow.

Personal projects are typically:
- Side projects
- Learning explorations

- Experimental ideas

## Folder Structure

- `notes/` - Session notes, ideas, learnings
- `progress/` - Session tracking for continuity
- `.agent/` - AI provider configuration
- `PROJECT.md` - This file
- `DECISIONS.md` - Key architectural decisions

## Getting Started

1. Review `.agent/config.json` to see how provider is configured
2. Check `DECISIONS.md` for key decisions made
3. Look at `progress/` to understand session tracking
4. Check `notes/` for session summaries

## Provider Configuration

See `.agent/config.json` for your chosen provider setup.

**To switch providers:**
1. Update `.agent/config.json` - change the `"provider"` field
2. Run the corresponding setup script (if first time)
3. Continue working - no workflow changes needed
